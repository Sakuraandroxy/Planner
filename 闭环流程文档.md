# AirSim 闭环演示：流程文档

> 本文档说明如何从零开始搭建一个基于 VLM（视觉语言模型）的无人机闭环导航系统。
> 核心思路：不需要本地跑任何深度学习模型，只需调 VLM API 就能实现零样本规划。

---

## 一、整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                      你的控制脚本                            │
│               run_airsim_closed_loop.py                     │
│                                                             │
│  ┌──────────┐    ┌──────────────┐    ┌──────────────────┐  │
│  │ AirSim   │───>│ VLMPlanner   │───>│ VLM API          │  │
│  │ 仿真器   │    │ (编码图片)    │    │ (GPT-4o/Qwen等)  │  │
│  └──────────┘    └──────────────┘    └──────────────────┘  │
│       │                 │                                      │
│       │                 ▼                                      │
│       │          ┌──────────────┐                              │
│       └──────────│ 动作 → 航点   │                              │
│                  │ 执行飞行      │                              │
│                  └──────────────┘                              │
└─────────────────────────────────────────────────────────────┘
```

**不依赖论文源码**（因为论文源码是为训练/评估设计的，依赖 torch/transformers）。
**只依赖：** `airsim`（官方 Python API）+ `openai` + `Pillow` + `numpy`。

## 二、前置条件

### 2.1 安装 AirSim

AirSim 是微软开源的无人机/汽车仿真器，基于 Unreal Engine。

- **下载**：从 [AirSim Releases](https://github.com/Microsoft/AirSim/releases) 下载预编译的 Windows 二进制包（如 `Blocks.zip`、`Neighborhood.zip`）
- **解压**到任意目录
- **启动**：双击 `Blocks.exe` 或 `Neighborhood.exe`，等 Unreal 引擎加载完成

### 2.2 安装 AirSim Python API

```bash
pip install airsim
```

这个包是 AirSim 官方提供的 Python 客户端，通过 RPC 协议与 AirSim 通信。  
**跟论文源码的 `airsim_plugin/AirVLNSimulatorClientTool.py` 功能相同**，但更简单直接。

### 2.3 配置 VLM API

设置环境变量：

```powershell
# OpenAI
set PLANNER_BASE_URL=https://api.openai.com/v1
set PLANNER_API_KEY=sk-xxx
set PLANNER_MODEL_NAME=gpt-4o

# 或任意 OpenAI 兼容 API（如 ModelScope、vLLM、Ollama 等）
set PLANNER_BASE_URL=https://your-endpoint/v1
set PLANNER_API_KEY=your-key
set PLANNER_MODEL_NAME=Qwen2.5-VL-72B
```

### 2.4 安装其他依赖

```bash
pip install openai pillow numpy
```

## 三、闭环流程详解

### Step 1: 连接 AirSim

```python
import airsim
client = airsim.MultirotorClient()
client.confirmConnection()  # 确认连接成功
```

AirSim 默认在本机的 41451 端口开 RPC 服务。`MultirotorClient()` 会自动连到这个端口。

### Step 2: 获取无人机当前帧

```python
responses = client.simGetImages([
    airsim.ImageRequest("front_center", airsim.ImageType.Scene, False, False)
])
# responses[0] 包含图片的 RGB 字节数据
img_bytes = responses[0].image_data_uint8
# 转成 numpy/PIL 格式送给 VLM
```

AirSim 返回的是原始 RGB 字节数组，需要 reshape 成 `(height, width, 3)` 的格式。

### Step 3: 调用 VLMPlanner 生成动作序列

```python
from planner.vlm_planner import VLMPlanner
from planner.config import PlannerConfig

cfg = PlannerConfig(
    base_url=os.environ["PLANNER_BASE_URL"],
    api_key=os.environ["PLANNER_API_KEY"],
    model_name=os.environ["PLANNER_MODEL_NAME"],
    candidate_count=5,
)
planner = VLMPlanner(cfg)

trajectories, scene_analysis, raw_json, reasoning = planner.generate_candidates(
    frame,  # PIL Image 或 numpy array
    "向前飞行，到达红色屋顶的建筑上方"  # 任务描述
)
```

`generate_candidates` 内部做了：
1. 将图片 base64 编码
2. 构建 system prompt + user prompt（含图片和任务描述）
3. 调用 OpenAI 格式的 API
4. 解析返回的 JSON，提取候选动作序列
5. 计算每条候选轨迹的 dx, dy, dz, dphi（终点位移）

### Step 4: 将动作序列转成 AirSim 航点

VLM 返回的是原子动作：`["forward", "forward", "left", "forward", ...]`

需要转成世界坐标下的航点：

```python
def actions_to_waypoints(actions, current_pos, current_yaw):
    """原子动作 -> 世界坐标航点列表"""
    x, y, z = current_pos
    yaw = current_yaw
    waypoints = []
    for action in actions:
        if action == "forward":
            rad = math.radians(yaw)
            x += 0.1 * math.cos(rad)
            y += 0.1 * math.sin(rad)
        elif action == "left":
            yaw -= 15.0
        elif action == "right":
            yaw += 15.0
        elif action == "up":
            z -= 0.1
        elif action == "down":
            z += 0.1
        waypoints.append([x, y, z])
    return waypoints
```

### Step 5: 让 AirSim 执行航点

```python
# 获取当前位置
pose = client.simGetVehiclePose()
current_pos = [pose.position.x_val, pose.position.y_val, pose.position.z_val]

# 转成航点
waypoints = actions_to_waypoints(best_actions, current_pos, current_yaw)

# 让无人机飞
for wp in waypoints:
    client.moveToPositionAsync(wp[0], wp[1], wp[2], velocity=2.0).join()
```

### Step 6: 闭环循环

```
while 未到达目标 and 步数 < 最大步数:
    1. simGetImages() → 获取当前帧
    2. VLMPlanner.generate_candidates() → 得到动作序列
    3. actions_to_waypoints() → 转航点
    4. moveToPositionAsync() → 执行
    5. simGetVehiclePose() → 获取新位置
    6. 计算距目标距离，判断是否完成
```

## 四、论文源码 vs 最小化方案对比

| 功能 | 论文源码做法 | 我们最小化方案 |
|------|------------|--------------|
| AirSim 连接 | `AirVLNSimulatorClientTool`（自定义 TCP 服务） | `airsim.MultirotorClient()`（官方 API） |
| 环境管理 | `AirVLNENV`（batch=8 多场景管理） | 直接 `client.simGetImages()` |
| 状态追踪 | `EvalBatchState`（多 episode 并行） | 简单变量记录位置和步数 |
| 模型调用 | `TravelModelWrapper`（加载本地 torch 模型） | `VLMPlanner.generate_candidates()`（调 API） |
| 目标检测 | `DinoMonitor`（本地 DINO 模型） | VLM 自身判断（prompt 中描述目标） |
| 航点优化 | `TrajModel`（本地轨迹优化模型） | 直接动作序列转航点 |
| 数据集 | TravelUAV 数据集（训练/评测用） | 不需要，只需 AirSim 地图 |
| 依赖 | torch, transformers, detectron2 等 | openai, pillow, numpy, airsim |

### 什么时候用论文源码

- **跑完整评测**：需要加载 TravelUAV 数据集、评测指标（NE/SR）
- **训练模型**：需要 torch + GPU 训练 LLM/Traj 模型
- **多场景并行评测**：batch=8 同时跑多个 AirSim 实例

### 什么时候用最小化方案（我们）

- **快速验证**：只想看 VLM 能不能规划出合理的轨迹
- **演示/调试**：不需要数据集，打开 AirSim 就能跑
- **接入真实无人机**：把 `moveToPositionAsync` 换成 ROS/串口指令即可

## 五、快速开始

### 5.1 简易演示（不需要 AirSim）

```powershell
pip install openai pillow numpy
set PLANNER_BASE_URL=https://api.openai.com/v1
set PLANNER_API_KEY=sk-xxx
set PLANNER_MODEL_NAME=gpt-4o

python run_airsim_closed_loop.py --demo
```

会生成一张模拟的纯色天空图，发给 VLM，输出候选轨迹。

### 5.2 AirSim 闭环演示（需要 AirSim 运行）

```powershell
# 终端1：启动 AirSim（双击 Blocks.exe）
# 终端2：
set PLANNER_BASE_URL=https://api.openai.com/v1
set PLANNER_API_KEY=sk-xxx
set PLANNER_MODEL_NAME=gpt-4o
set task="向前飞到红色汽车上方"

python run_airsim_closed_loop.py --airsim
```

这个模式会：
1. 连接 AirSim
2. 获取当前帧
3. 送给 VLM 得到动作序列
4. 让无人机执行
5. 循环直到到达目标
