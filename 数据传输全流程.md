# Uni-LaViRA 数据传输全流程

## 系统架构总览

三个核心组件：AirSim 仿真器、Python 规划器、VLM API

## 各模块职责

### AirSim 仿真器 (Blocks.exe)
- 基于 Unreal Engine 4 的无人机物理仿真环境
- 渲染 3D 世界，模拟飞行物理
- 通过 msgpack-rpc 协议（端口 41451）与外部通信

### Python 规划器主循环 (后台线程)
- 连接 AirSim → 解锁 → 起飞
- 任务空时悬停，不调 VLM
- 拍照 → 调 VLM → 解析动作 → 执行 → 循环

### VLM Planner (planner/vlm_planner.py)
- RGB + Depth 图像 base64 编码
- 构建提示词，调用 VLM API
- 解析 JSON，提取候选轨迹、选中索引、完成标志

### Flask Web 服务器 (主线程)
- 依次提供：静态文件、SSE 实时推送、图片端点、任务更新

### 前端仪表盘 (frontend/)
- HTML/CSS/JS 静态文件
- EventSource (SSE) 实时接收状态
- 显示：无人机画面、深度图、位置、VLM 推理、候选轨迹

## 通信协议详解

### 1. AirSim ↔ Python (msgpack-rpc)
- 端口 41451，自动连接
- 主要接口：simGetImages、simGetVehiclePose、moveToPositionAsync、rotateToYawAsync、takeoffAsync、simGetCollisionInfo、enableApiControl、armDisarm

### 2. Python ↔ VLM API (HTTP/REST)
- OpenAI 兼容协议，POST /v1/chat/completions
- 请求体：model + messages (system + user with images + text)
- 响应：含候选轨迹、selected_index、done 标志的 JSON

### 3. Python ↔ Web 前端 (HTTP + SSE)
- / events (SSE)：实时推送 SharedState
- / frame、/depth_frame (GET)：返回当前图像
- / task (POST)：更新任务目标

## 完整闭环数据流

### 步骤时序
1. 连接 AirSim → 解锁 → 起飞
2. 如果任务为空：悬停，等待用户输入
3. 用户设置任务后：simGetImages(RGB+Depth) → base64 编码
4. POST VLM API → 得到候选轨迹
5. 解析 selected_index → 选中最优轨迹
6. rotateToYawAsync (rotate) → moveToPositionAsync (move)
7. simGetVehiclePose → 更新位姿
8. 回到步骤 3，直到 VLM 返回 done=true
9. 降落、锁定

## 开发者备注

- 图片传输：VLM 接收原始分辨率，前端显示原始分辨率
- 任务控制：任务空时不调 VLM，无人机悬停
- 线程安全：SharedState 使用 threading.Lock 保护
- 未来扩展：可将 trajectories[selected_index] 替换为世界模型打分逻辑
