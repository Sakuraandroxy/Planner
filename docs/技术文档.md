# Uni-LaViRA 规划器闭环流程与核心代码解析

本文档系统梳理了本项目中基于大视觉语言模型（VLM）的无人机闭环导航系统的数据传输流程，并对核心代码文件的功能进行了详细拆解。

---

## 一、 闭环数据传输流程（Data Flow）

闭环导航的核心是“**感知 -> 思考 -> 执行 -> 再感知**”的循环。在一个完整的控制时间步（Step）中，数据的流转顺序如下：

```text
[AirSim 仿真器] --(RGB / 深度图)--> [run_airsim_web.py (后台线程)]
                                            |
                         +------------------+------------------+
                         | (写入共享状态)                        | (触发规划)
                         v                                     v
                 [web_dashboard.py]                    [vlm_planner.py]
              (更新并推送网页画面与日志)                 (构建 Prompt 并调用 VLM)
                         |                                     |
                         v                                     v
               [浏览器前端 Dashboard]                      [VLM API 大脑]
               (用户下发任务 / 观测状态)                   (返回 JSON 规划文本)
                         ^                                     |
                         | (输入新任务)                         v
                 [run_airsim_web.py] <---(动作解析与 Delta 计算)-+
                         |
                 (调用 trajectory.py 转化为航点)
                         |
                         v
           [AirSim 仿真器执行物理移动] ---> 进入下一个 Step 循环
```

### 详细步骤分解：

1. **传感器数据采集与推送**：
   * `run_airsim_web.py` 通过后台图像流线程 `_start_frame_stream` 以固定间隔抓取 AirSim 仿真器中的 RGB 彩色图与 Depth 深度图。
   * 图像字节流被写入线程安全的共享状态类 `SharedState`（位于 `web_dashboard.py`）。
   * `web_dashboard.py` 启动的 Flask 服务器读取该状态，将图像以 MJPEG 视频流和 Server-Sent Events (SSE) 事件的形式推送到前端浏览器。

2. **触发大模型规划**：
   * 主循环 `main_loop` 拿到当前的图像帧和用户在网页端输入的文本任务指令（`cur_task`），调用 `VLMPlanner.generate_candidates()`（位于 `vlm_planner.py`）。
   * `vlm_planner.py` 将图像转化为 Base64 编码，并从 `prompt_templates.py` 中读取系统提示词模板，组合填入任务目标、单步运动跨度等参数，打包成 API 请求发送给 VLM。

3. **大模型决策与解析**：
   * VLM 评估当前视觉场景后，返回一段包含场景分析和多条候选轨迹的 JSON 格式文本。
   * `vlm_planner.py` 的解析模块利用正则表达式提取 JSON，在发生截断时执行基础的括号补齐（容错处理），最终输出结构化的候选轨迹列表。

4. **动作到物理位移的转化**：
   * 选定的最佳轨迹（一串符号动作，如 `["forward", "left", "forward"]`）被送入 `trajectory.py` 的 `compute_delta()` 函数。
   * 结合 `config.py` 中的物理步长，该函数通过三角几何计算，推算出该轨迹在无人机局部坐标系下导致的相对位移和偏航角变化（Delta_x, Delta_y, Delta_z, Delta_yaw）。

5. **底层控制执行**：
   * `run_airsim_web.py` 将相对位移叠加到无人机当前坐标上，生成一组绝对航点。
   * 依次调用 AirSim 运动接口控制无人机旋转（`rotateToYawAsync`）和平移（`moveToPositionAsync`）。
   * 控制完成后，进行碰撞检测与避障恢复，并将最新的状态写回共享空间，页面更新，进入下一个 Step。

## 二、 核心代码文件作用解析

### 1. `config.py`（系统配置参数）
* **作用**：集中管理整个规划系统的运行常数。
* **核心内容**：
  * **API 访问配置**：大模型接口的 `base_url`、`api_key` 和模型名称。
  * **物理运动参数**：定义每个符号动作在仿真器里的物理步长（如 `horizontal_step` 前进 5 米，`yaw_step_deg` 旋转 15 度），控制执行粒度。
  * **仿真与噪声参数**：规定模拟传感器误差的扰动高斯标准差（`noise_sigma`），用于提高控制闭环对噪声的容忍能力。

### 2. `trajectory.py`（运动几何学转换）
* **作用**：提供符号化原子动作（如 `left`、`forward`）到物理三维位移量（相对坐标）的几何映射。
* **核心内容**：
  * **原子动作定义**：定义 `forward`、`backward`、`left`、`right`、`up`、`down` 六类枚举。
  * **`compute_delta` 函数**：累加动作列表的物理效应。例如当累加 `forward` 时，利用当前的偏航角变化 yaw 计算水平投影坐标增量：
    * dx += step * cos(rad(yaw))
    * dy += step * sin(rad(yaw))
  * **`add_noise` 函数**：在无噪声位移（`clean_delta`）上加入轻微的高斯随机噪声，模拟物理执行器的不确定性。

### 3. `prompt_templates.py`（提示词工程模板）
* **作用**：定义了大模型扮演的角色行为、输入输出规约，是 Zero-Shot 规划质量的核心。
* **核心内容**：
  * **动作与环境映射解释**：向大模型告知可用的动作空间、步长大小，以及深度图的冷暖色所代表的远近关系（暖色近、冷色远）。
  * **输出 Schema 约束**：强制模型返回 JSON 结构。结构中必须包含当前的场景分析（`scene_analysis`）、全局推理总结（`reasoning_summary`）、候选路径列表（`candidates`）以及任务终止信号（`done`）。
  * **JSON 防破损引导**：在提示词中强硬规定提示词和摘要中禁止在推理中使用双引号 `"`，以单引号或括号代替，以防其破坏 JSON 的键值对闭合。

### 4. `vlm_planner.py`（大模型接口适配器）
* **作用**：负责图像编码、大模型 API 通信以及非结构化输出的结构化转换。
* **核心内容**：
  * **Base64 图像转换**：将 OpenCV 格式或 PIL 格式的 RGB 与 Depth 帧压缩编码，作为多模态消息的内容发送。
  * **`_parse_vlm_response` JSON 格式容错解析**：利用正则清洗 Markdown 代码块，在遇到 JSON 破损（例如大模型响应因 token 耗尽而被截断）时，通过匹配花括号 `{}` 和方括号 `[]` 的开启数量，在尾部自动补齐闭合符号，降低系统解析出错率。

### 5. `web_dashboard.py`（交互网关与共享状态）
* **作用**：提供前台交互 Web 界面和多线程状态总线。
* **核心内容**：
  * **`SharedState` 共享类**：基于线程锁 `threading.Lock` 实现，用于存放当前的传感器帧、无人机位姿、碰撞状态及大模型推理过程。控制线程和 Web 服务器线程以此作为唯一数据交换点。
  * **视频流网关**：基于 HTTP MJPEG（`multipart/x-mixed-replace`）实现无需额外插件的实时第一人称画面直播。
  * **事件流网关**：基于 SSE（Server-Sent Events）技术实现将后台规划数据（如大模型当前的思考步骤）低延迟推送到前端渲染展示。

### 6. `run_airsim_web.py`（闭环调度主程序）
* **作用**：系统的调度中心与物理控制器。
* **核心内容**：
  * **多线程架构设计**：拉起 Flask 网页服务器作为主进程，同时开启后台图像抓取线程，并在新线程中运行整个闭环控制主循环（`main_loop`）。
  * **符号动作到物理航点映射**：解析轨迹并将其转换为仿真器可理解的世界三维坐标航点序列。
  * **平移与旋转分离执行**：自动将轨迹中的动作分组。旋转动作（`left`/`right`）转交给 `rotateToYawAsync` 串行执行，平移动作转交给 `moveToPositionAsync` 协同执行。
  * **碰撞避障策略**：实时监控传感器碰撞反馈，一旦发生碰撞强制向后方平退一定跨度，以保障机器人在无人介入情况下的基础安全。
