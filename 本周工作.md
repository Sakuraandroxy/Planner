# 本周工作：Uni-LaViRA AirSim 闭环导航

## 1. 本周核心进展

本周主要完成了 AirSim 场景下的 VLM 闭环导航原型：无人机能够从 Web 页面接收自然语言任务，实时获取 RGB 图和深度信息，调用多模态大模型生成动作，并在 AirSim 中执行前进、旋转、升降等控制指令。

核心目标从“能调模型给动作”推进到“模型、深度、执行、安全约束形成闭环”。

## 2. 系统闭环已打通

当前主流程如下：

```text
Web 输入任务
  -> AirSim 采集 RGB 图 / 深度图 / 位姿
  -> VLM 分析场景并输出候选动作
  -> 解析 JSON 动作
  -> 转换为 AirSim 航点
  -> 执行动作
  -> 下一帧重新观察并规划
```

本周完成的关键模块包括：

| 模块 | 本周完成内容 |
|---|---|
| `run_airsim_web.py` | 打通 AirSim、Planner、Web Dashboard 的主循环 |
| `sim/airsim_client.py` | 封装 RGB 图、深度图、原始深度矩阵、位姿和控制接口 |
| `agent/planner.py` | 组织 VLM 调用、目标深度估计、轨迹生成和上下文输入 |
| `agent/response_parser.py` | 解析 VLM JSON，并对不安全 forward 动作做裁剪 |
| `planner/trajectory.py` | 将动作序列转换为相对位移和航点执行参数 |
| `web/` | 实时显示画面、深度图、状态、候选动作和推理结果 |

## 3. 深度估计方案迭代

本周重点解决了“VLM 是否能知道目标深度”的问题。

最初方案是把深度图直接作为图片传给 VLM，让模型自己读深度；实践发现这种方式不稳定，模型容易读错、猜错或把近处物体当作目标。

当前采用的新方案是：

```text
VLM 负责找目标 bbox
Python 负责从 AirSim 原始 DepthPerspective 矩阵中取真实深度
再把目标深度作为文本提示传回规划模型
```

主要改动：

- 新增 `agent/target_depth.py`，让 VLM 输出目标框 `bbox_norm`。
- 新增 `get_depth_meters()`，直接读取 AirSim 原始米制深度矩阵。
- 将 RGB bbox 映射到深度矩阵坐标，计算目标区域深度。
- 对“较远汽车”这类任务，支持多个候选目标，并选择深度更大的目标。
- 增加调试日志，打印 `raw_bbox`、`rgb_bbox`、`depth_bbox`、`median/min/mean`。

目前已经确认：目标框检测和深度矩阵采样链路可以跑通，但远距离小目标仍会受深度图分辨率影响，后续需要继续优化采样策略。

## 4. 飞行安全约束

本周发现一个重要问题：当目标深度估计偏大时，VLM 可能输出一次性长距离前进，例如 `forward 100`，这对闭环导航不安全。

因此新增了单次最大前进距离约束：

- 配置项：`max_forward_step`
- 环境变量：`PLANNER_MAX_FORWARD_STEP`
- 默认值：`10m`

现在即使模型输出很大的前进距离，程序也会裁剪到安全范围内。这样无人机不会一次性飞到远处，而是采用“前进一步、重新观察、再规划”的闭环策略。

## 5. 当前问题

当前系统已经可以完成基础闭环，但仍存在几个关键问题：

1. **远目标深度仍不稳定**  
   AirSim RGB 图为高分辨率，而深度矩阵分辨率较低。远处车辆映射到深度图后可能只剩几个像素，容易采到背景深度。

2. **bbox 依赖 MLLM 输出质量**  
   部分模型会输出分析过程、占位符或不合法 JSON，需要继续增强结构化输出稳定性。

3. **单帧决策容易误识别目标**  
   如果当前帧刚好看不到原目标，VLM 可能把画面中的另一个相似物体当成目标。

4. **任务完成判断还不够可靠**  
   目前主要依赖 VLM 和当前帧，没有充分利用历史深度、已执行距离和目标跟踪状态。

## 6. 未来计划

### 6.1 解决单帧视觉不可靠问题

后续需要引入目标状态记忆，避免每帧都重新相信 VLM 的单帧判断。

计划增加一个 `TargetTracker`，维护以下状态：

- 当前锁定目标类别、颜色、bbox 和历史深度
- 初始目标深度
- 最近一次有效目标深度
- 已执行 forward 总距离
- 目标连续丢失帧数
- 当前状态：`SEARCHING / TRACKING / NEAR_TARGET / ARRIVED / LOST_TARGET`

核心策略：

- 第一次发现目标后锁定目标，后续不允许随意切换到另一个相似目标。
- 如果目标短暂消失，不立刻追新目标，而是结合历史距离判断是否已经到达。
- 当目标深度小于阈值，或累计前进距离接近初始目标距离时，优先进入 `NEAR_TARGET / ARRIVED`。
- 只有长时间丢失且明显未到达时，才重新搜索目标。

这样可以解决“无人机已经到达目标附近，但当前帧没拍到目标，VLM 又发现假目标”的问题。

### 6.2 优化远距离目标深度估计

- 提高或对齐 AirSim RGB 与深度图分辨率。
- 对小 bbox 使用扩张采样和近分位数深度，例如 `p10 / p20`，减少背景深度干扰。
- 在日志中增加更多深度统计，辅助判断采样是否落在目标本体上。

### 6.3 强化结构化输出稳定性

- 将 bbox 检测输出和主规划输出进一步规范化。
- 对 VLM 返回的 JSON 增加更强的校验与自动修复。
- 必要时把目标检测与规划合并到一次模型调用中，减少模型跑偏。

### 6.4 增加任务完成判定

- 引入到达半径，例如 `arrival_distance_threshold=3m`。
- 结合目标深度、累计前进距离、目标丢失帧数和 VLM 判断共同决定任务是否完成。
- 到达后自动悬停并清空任务，避免继续追逐其他目标。

## 7. 本周结论

本周的主要成果是：完成了 AirSim-VLM-Web 的闭环导航原型，并把深度信息从“模型看图猜距离”升级为“程序基于原始深度矩阵计算距离”。

当前系统已经具备基本自主导航能力，但要进一步可靠运行，还需要重点解决目标跟踪、远距离深度采样和任务完成判定问题。
